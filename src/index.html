Matrix Chain Multiplication asks: given n matrices A₁, A₂, …, Aₙ, in what order should we parenthesize the product to minimize scalar multiplications? Matrix multiplication is associative, so the order matters only for cost.

Optimal Substructure
Any optimal parenthesization of Ai..j must split at some k where i ≤ k < j, and the two sub-products Ai..k and Ak+1..j must themselves be optimally parenthesized.
DP Table
m[i][j] stores the minimum cost to multiply matrices i through j. Fill order: by chain length ℓ = 1, 2, …, n. m[i][i] = 0 for all i (base case).
O(n³) Complexity
Three nested loops: chain length, start index, split point. Space is O(n²) for the m and s tables. Reconstructing the optimal order from s takes O(n).

Educational Overview Work, span, and parallelism analysis
›

Parallel Merge Sort extends classic merge sort using a fork-join model. Each recursive call spawns two subtasks that run concurrently, then syncs before merging. On a machine with unlimited processors it achieves polylogarithmic span.

Work vs Span
Work T₁ = O(n log n) — total operations as if sequential.
Span T∞ = O(log² n) — critical path length with infinite processors.
Parallelism
Parallelism = Work / Span = O(n log n / log² n) = O(n / log n). This means we can usefully employ up to O(n / log n) processors before returns diminish.
Amdahl's Law
Sequential fraction of the merge step limits speedup. A parallel merge (e.g. binary-search merge) reduces span to O(log n), improving parallelism to O(n).

The Euclidean Algorithm (≈300 BC) computes gcd(a, b) by repeatedly replacing (a, b) with (b, a mod b) until the remainder is zero. It runs in O(log min(a, b)) divisions — one of the oldest known algorithms.

Bézout's Identity
For any integers a and b there exist x, y such that ax + by = gcd(a, b). The extended algorithm finds x and y via back-substitution.
Modular Inverse
If gcd(a, n) = 1 then a⁻¹ mod n = x mod n where ax + ny = 1. This is the foundation of RSA and Diffie-Hellman.
Lamé's Theorem — Worst Case
Divisions never exceed 5 × log₁₀(min(a, b)). Worst case is consecutive Fibonacci numbers — try the preset above.

Worked Example — gcd(35, 15):
35 = 2·15 + 5  →  15 = 3·5 + 0  →  gcd = 5
Back-sub: 5 = 35 − 2·15  ∴  x = 1, y = −2
Verify: 35(1) + 15(−2) = 5 ✓
